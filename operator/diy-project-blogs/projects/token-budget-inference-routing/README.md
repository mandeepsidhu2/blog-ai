# Token Budget Inference Routing

This internal project measures how simple routing policies trade cost, latency,
and quality risk across a small set of LLM application tasks.

The run is deterministic and does not use local model inference or torch. The
goal is not to benchmark a provider. It is to make routing policy failure modes
inspectable before public article drafting.

Files:

- `dataset.json`: twelve task traces with token budgets, SLA targets, context
  reuse, risk, and expected serving tier.
- `run-experiment.mjs`: policy simulator and metric generator.
- `results.json`: detailed output from the latest run.
- `output.txt`: concise terminal output used in article drafting.
- `chart.svg`: scorecard visual generated by the script.

Run:

```sh
node operator/diy-project-blogs/projects/token-budget-inference-routing/run-experiment.mjs
```

Use the Codex-bundled Node runtime if `node` is unavailable on PATH.
